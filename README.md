# Robust Partial-Window SoH Estimation for Lithium-Ion Batteries

This repository contains the public code and result artifacts for the manuscript:

**Robust Partial-Window State-of-Health Estimation for Lithium-Ion Batteries Using Chemistry-Aware Multi-Health Indicators and Missing-Aware Ensembles**

The project evaluates lithium-ion battery state-of-health (SoH) estimation under partial-charging conditions. It compares controlled single-HI baselines, approximate Kadem-style GPR/EKF baselines, fixed-window multi-HI tabular models, and Top-K Missing-Aware Window-Gated Multi-Model Ensemble variants (TopK-MAW-GME-K3/K5).

## What is included

- Sanitized Google Colab notebooks for Dataset 1, Dataset 2, Dataset 3, BMS replay, runtime ablation, and Kadem-style GPR/EKF baseline experiments.
- Extracted multi-HI feature tables used by the executed notebooks.
- CSV/XLSX result files used to prepare the manuscript tables and figures.
- Figure files generated by the experiment notebooks.
- CPU-only Google Colab environment report.
- Helper script to regenerate compact manuscript summary tables from the stored result files.

Notebook outputs and Colab execution metadata were stripped before packaging so that the repository is suitable for public GitHub upload. The numerical result files are kept separately under `results/`.


The helper script reads the committed CSV results and writes compact summary files into `outputs/`.

## Main result summary

Under leave-one-cell-out validation, the best fixed-window multi-HI models achieved the following RMSE values in SoH-fraction units:

| Dataset | Best controlled single-HI RMSE | Controlled approx. GPR/EKF RMSE | Best fixed-window Multi-HI RMSE |
|---|---:|---:|---:|
| Dataset 1 NMC/UofM | 0.0175 | 0.0194 | 0.0104 |
| Dataset 2 Oxford | 0.0085 | 0.0127 | 0.0043 |
| Dataset 3 LFP/graphite | 0.0117 | 0.0156 | 0.0023 |

TopK-MAW-GME-K3/K5 are intended as missing-window robustness modes rather than as the absolute lowest-latency estimator in every setting. The BMS-oriented stress tests evaluate update continuity when the best single fixed window is unavailable.
